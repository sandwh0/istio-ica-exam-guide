# Installation and Upgrades

This topic covers how Istio is installed, customised, changed, and upgraded. For the exam, you need to be comfortable with both the mechanics of the installation method and the effect that your changes have on the running control plane and workloads.

## For the exam you should be confident with the following

1. Install Istio using `istioctl` and Helm.
   - Be familiar with custom configuration and different installation modes.
2. Edit an existing Istio installation using custom configuration with `istioctl` and Helm.
3. Understand how to use the `IstioOperator` resource.
4. Upgrade Istio using a canary upgrade approach.
5. Inject sidecars at namespace level and manually inject individual workloads.
